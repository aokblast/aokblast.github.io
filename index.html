What keyword "inline" means in C

Posted on 2021-06-10 Edited on 2021-06-11 In Programming Language

| Reference: http://www.greenend.org.uk/rjk/tech/inline.html

Recently, many friends ask me what the keyword “inline” means in C and if it is necessary to be a static function when we put inline in the function definition?

History

The keyword “inline” was first introduced in GNU C and it took a long time to be a part of C standard in C99 and after version.
Actually, there are some difference of inline between GNU C and standard C.

How inline is used in C

  1. static inline
  2. inline
  3. extern inline

What is the target inline wants to achieve

As we know, every time when we call a function, it takes effort for the computer to do some preparation(like push some registers’ status to the stack) which is sometimes unnecessary and cause performance issue.
Therefore, to solve this problem, GNU C introduce a keyword “inline” to suggest compiler to expand the whole function on the location where it was called.
Namely, the function which is successfully inlined don’t have to make redundant preparation to execute.

When a inline function actually not inline

It determine if the compiler expand the code directly.

  1. Waste space
  2. Function pointer
  3. Effect the performance of instruction cache

Inline in Standard C

  1. static inline:
    Static + inline.
    It is static so it can only be used in the same c source file. Namely, it cannot be called by other c source.
    It is inline so it will be expanded on the original location when it can, if cannot, it will generate assambly code and do a normal call.
  2. inline:
    It always inline the function and will not generate assembly code for the original function.
    So it may fail to compile when there are situations cannot be inlined.
    You can consider it is a macro and expand the function directly.
  3. extern inline:
    Extern + inline.
    Extern means it can be called by other function and always generate assembly code for the function.(Function call in other source will not be inlined)
    It is inline so it will be expanded on the original location when it can, if cannot, it will call the function which compiler generate.

Inline in GNU C

  1. static inline:
    Same as Standard C.
  2. inline(Act like extern inline of Standard C):
    It always generate assembly code for the function to make other c source file can call.(Function call in other source will not be inlined)
    GCC will inlined the function if it can, if cannot, call the original function.
  3. extern inline(Act like inline of Standard C):
    Compiler always not generate assembly code for the function and expand all of the function call.

Experiment

I really do some experiment but I won’t show the result one by one. Because in different optimizing flag and different method to use inline may cause different result.

Sometimes compiler may not generate assembly code when analyzing a function but when the function was called, compiler doesn’t think it can be inlined and discovered that the function cannot be called. Therefore, it cause compile error because compiler cannot find the function reference.

If you are interesting in this, you can perform experiment by yourself with different optimizing flag and different method.
